# Evidence Boundary

This repo works best when each layer keeps its own shape and job inside the stack.

## Interaction Artifacts

Concrete session transcripts, prompts, repo outputs, and generated reports.

## Phenomenology

Reported feel, resonance, intensity, or perceived significance of a session.

## Ontology

Conceptual language used to describe recurring configurations, including attractor-state framing.

## Independent Evidence

Artifacts that can be re-run, inspected, or audited without relying on the original interpretation.

## Rule

Keep ontology and phenomenology named as such, and let interaction artifacts and rerunnable outputs carry the measured lane.

The code here contributes most directly to interaction-artifact analysis and to formatting evidence that can be rerun, inspected, and compared across sessions.
